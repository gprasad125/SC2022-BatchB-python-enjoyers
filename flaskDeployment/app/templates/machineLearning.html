Machine Learning

K-Nearest Neighbor

A K-Nearest Neighbor or KNN model works by finding the distance between the datapoint to be classified and some k number of example datapoints. It then uses majority rules to determine the classification of the query.

We used this model as a sort of default model to start the modeling process. As for all the models, we split the data, then initialized it with the optimized parameters estimated by GridSearchCV. The model had an accuracy of about 60%. The low accuracy of this model makes sense given the visualizations we made, because there was no clear distinction between potable and non-potable data-points in regards to where they were on a graph.

Decision Tree Classifier

A Decision Tree Classifier uses conditions, represented by internal nodes, and leaf nodes which represent the output based on these conditions. Depending on its depth, the query data point will iterate through many conditions before being classified finally.

We chose this model in consideration of a parallel coordinates plot we made, which showed that potability was more likely given certain ranges for different columns. This model had a slightly improved accuracy from the KNN model; around 66%.

Random Forest Classifier Explanation

The Random Forest Classifier is basically an extended version of the Decision Tree Classifier, requiring the same input. It utilizes multiple Decision Trees (hence, a forest) for each subset in the data, while using averaging to improve the predictive accuracy and to combat over-fitting.

In the end, the Random Forest Classifier had a accuracy of roughly 70%, our most accurate model so far.
