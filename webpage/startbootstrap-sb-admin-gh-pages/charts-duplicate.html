We used this model as sort of a default model to start the modeling process. We split the data, then initialized it with the optimized parameters estimated by GridSearchCV. The model had an accuracy of about 60%, and a mean squared error of around 0.407. Itâ€™s precision was around .17, recall around .42, and an F-measure around .24. Decision Tree Classifier: Explanation

We chose this model in consideration of a parallel coordinates plot we made, which showed that potability was more likely given certain ranges for different columns. The process was very similar to the K-Nearest Neighbors model; we started by splitting the data, and then imported and initialized the model using the optimized hyperparameters found using GridSearchCV. This model had a slightly improved accuracy from the KNN model with about 66%, but the precision, recall, and F-measure are skewed. This is the likely result of the uneven spread of data, (we have 1200 non-potable data-points and only 811 potable ones), which may also be contributing to lower accuracy.

Random Forest Classifier: Explanation

The Random Forest Classifier is basically an extended version of the Decision Tree Classifier, requiring the same input. It also utilizes multiple decision trees for each subset in the data, while using averaging to improve the predictive accuracy and to combat over-fitting.In the end, the Random Forest Classifier had a accuracy of 70% which is our highest yet!
